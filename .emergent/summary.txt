<original_problem_statement>
The user wants to build a mobile application for booking DJs for events (Une appli de placement de dj pour des prestations événementielles).

The application should have three main user roles:
1.  **Event Organizers**: Can browse DJs, book them for events, and manage their bookings.
2.  **DJs**: Can create detailed profiles, showcase their work (including video links and social media), and accept bookings.
3.  **Admin**: Can oversee the platform and manage commissions.

**Product Requirements:**
-   **User Roles**: DJs, Event Organizers, and an Admin.
-   **DJ Profiles**: Should include details like music style, experience, and allow for adding performance videos and social media links.
-   **Event & DJ Discovery**: Users should be able to search and filter for DJs and events based on event type and music style.
-   **Booking System**: A complete flow for organizers to book DJs for their events.
-   **Messaging**: A feature for DJs and Organizers to communicate.
-   **Monetization**: A commission system where the platform takes a 15% cut on each transaction. The payment for the MVP can be simulated.
-   **Payment Flow**: DJs' earnings are held in a pending balance until the event organizer confirms the service (prestation) has been successfully completed.
-   **Admin Dashboard**: An interface for the admin to view transactions and commissions.
-   **UI/UX**: The user has requested a Neon Style theme with a dark background, neon colors, and animated neon lines in the background.
